WELCOME TO TEACHER CONSOLE

The ultimate platform for classroom management

What's happening
  1. MSAL Browser popup → login.microsoftonline.com
  2. Teacher signs in as teacher1@faronicsna.onmicrosoft.com
  3. idToken returned; email extracted from Entra claims
  4. Browser calls /api/classlink?op=teacher&email=…
  5. Serverless function maps email → ClassLink sourcedId
  6. Classes + student counts returned → dropdown populates

Production difference

In production, DFC serves classes from tbl_RosteredClass (pre-synced). In this demo the API route calls ClassLink live to show what DFC does during a sync job.

What's happening
  1. MSAL popup — sign in as a student account
  2. Email extracted from Entra idToken
  3. Browser calls /api/classlink?op=student&email=...

Architecture note

ClassLink token endpoint: /token with HTTP Basic auth — not /oauth2/token as ClassLink docs state (confirmed by live POC).

DFC's sync job calls ClassLink and stores results in tbl_RosteredClass and tbl_RosteredStudent. Teacher Console and Student Client only query the local DB — never ClassLink directly.
